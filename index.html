Word Hunt Solver

This C++ program implements a word search puzzle solver for a 4x4 grid of letters, where the goal is to find valid words formed by adjacent letters, using a Trie data structure for efficient word lookup. The grid is explored using a depth-first search (DFS) approach, allowing movement in all 8 possible directions (vertically, horizontally, and diagonally). The program starts by loading a dictionary from a file (words_alpha.txt) and inserts words of length greater than three into a Trie. Each word is built by recursively exploring adjacent cells, and valid words are stored in a list if they exist in the Trie. After all possible words are found, they are sorted and printed. The program also ensures that each word is at least 4 letters long and checks for bounds and previously visited cells to avoid revisiting the same cell. The search process begins with reading a 4x4 grid of letters from the user, followed by initiating the word search on the grid. This combination of Trie-based lookup and DFS ensures efficient and thorough word searching in the grid, while backtracking prevents unnecessary re-exploration of already visited paths.

Infinite Platformer

This code implements a simple infinite jumper game using the Pygame library. The game features a character (kitty) that jumps on platforms to ascend while the platforms scroll down as the character moves up. The character's movement is governed by gravity and jump physics, with horizontal control enabled using the A and D keys. Platforms are dynamically repositioned to simulate an infinite scrolling effect, resetting to the top of the screen once they move below the visible area. The game ends when the character falls off the bottom of the screen, displaying a "Game Over" message with an option to restart by pressing the space bar. The code uses collision detection to ensure the character can jump only when standing on a platform, and the display is updated each frame to render the character, platforms, and game state. The game loop manages the core logic, including user input, collision checks, movement updates, and rendering, while maintaining a consistent frame rate using Pygame's clock.
